# Customer Behaviour Analysis

## Project Overview
This project involves a comprehensive analysis of customer shopping behaviors to uncover trends in purchasing patterns, product performance, and customer loyalty. The project is broken down into three main phases:
1. **Data Preprocessing & ETL**: Using Python and Pandas to clean, transform, and load the raw dataset into a relational database.
2. **Exploratory Data Analysis (EDA)**: Using SQL (PostgreSQL) to query the database and answer key business questions regarding revenue, discounts, subscriptions, and customer segmentation.
3. **Data Visualization**: Using Power BI to create an interactive dashboard that brings the SQL insights to life, making it easy to track KPIs and customer trends at a glance.

## Tech Stack
* **Language**: Python 3, SQL
* **Libraries**: `pandas`, `sqlalchemy`, `psycopg2-binary`
* **Database**: PostgreSQL
* **Visualization**: Power BI
* **Environment**: Jupyter Notebook

## Dataset
The analysis is based on the `customer_shopping_behavior.csv` dataset, which includes 3,900 customer records. Key features include:
* **Demographics**: Age, Gender, Location
* **Purchase Details**: Item Purchased, Category, Purchase Amount, Size, Color, Season
* **Behavioral Metrics**: Review Rating, Subscription Status, Shipping Type, Discount Applied, Previous Purchases, Frequency of Purchases

## Data Cleaning & Preprocessing (Python)
The `customer_behaviour_analysis.ipynb` notebook handles the ETL (Extract, Transform, Load) pipeline:
* **Handling Missing Data**: Missing `Review Rating` values were successfully imputed using the median rating of their respective product categories.
* **Data Standardization**: Column names were converted to `snake_case` for easier SQL querying.
* **Feature Engineering**: 
  * Created an `age_group` column (Young, Adult, Middle-Age, Senior) using quantile-based discretization.
  * Mapped text-based purchase frequencies (e.g., 'Weekly', 'Fortnightly') into a numerical `purchase_frequency_days` column.
* **Dimensionality Reduction**: Identified and dropped the `promo_code_used` column as it was 100% correlated with the `discount_applied` column.
* **Database Loading**: Automated the export of the cleaned DataFrame directly into a PostgreSQL database table named `customer`.

## Exploratory Data Analysis (SQL)
The `customer_analysis.sql` script contains queries designed to extract actionable business insights. Key questions answered include:
* **Revenue Demographics**: Total revenue generated by male vs. female customers.
* **Discount Behavior**: Identifying customers who used a discount but still spent above the overall average.
* **Product Performance**: Finding the top 5 products with the highest average review ratings.
* **Shipping Logistics**: Comparing average purchase amounts between Standard and Express shipping.
* **Subscription Value**: Analyzing if subscribed customers spend more overall.
* **Discount Rates**: Uncovering which 5 products have the highest percentage of discount applications.
* **Customer Segmentation**: Categorizing customers into 'New' (1 purchase), 'Returning' (2-10 purchases), and 'Loyal' (11+ purchases) segments.
* **Category Leaders**: Identifying the top 3 most purchased products within each specific category.
* **Loyalty & Subscriptions**: Determining if repeat buyers (more than 5 previous purchases) are more likely to subscribe.
* **Age Demographics**: Calculating the total revenue contribution of each age group.

## Interactive Dashboard (Power BI)
